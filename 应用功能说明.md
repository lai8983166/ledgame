# LED Game 应用功能说明

## 游戏列表

### Simple 编辑器演示

点击“Simple 编辑器演示”进入 Simple 编辑器。

#### 帧

- 点击数字帧切换当前帧。
- 点击“+”新增帧。
- 点击 `P` 将当前帧复制到前一帧。
- 点击 `N` 将当前帧复制到后一帧。
- 点击 `=` 将当前帧复制到所有其他帧。
- 点击 `-` 删除当前帧。
- 点击导入或导出按钮，导入 JSON 帧或导出当前帧 JSON。
- 使用 `A`、`D` 键切换帧。
- 复制到已存在的帧会先弹出覆盖确认；取消后不会修改目标帧。

#### 基础信息

- 编辑名称、描述、备注、矩阵宽度、矩阵高度。
- 点击“全局配置”打开游戏全局配置，主要设置游戏中使用到的资源，比如音效，动画。

#### 矩阵编辑

- 在“画笔”中选择“新增对象”（按Q切换）后，左键点击空白格新增单格对象。
- 在“新增对象”模式下左键拖拽，可框选区域并创建多格对象。
- 选择“选择/移动”后，左键点击对象进行选择，拖拽对象进行移动，也可按方向键移动。
- 在矩阵上滚动鼠标滚轮调整矩阵缩放。
- 点击“全景”查看和编辑矩阵范围之外的对象；再次点击退出全景。
- 在全景模式中点击“选择”，再点击对象进行多选；点击“合并”合并选中的同色对象。
- 右键点击矩阵可打开选择或合并操作菜单。
- 重叠对象按 Simple 规则显示，绿色对象优先显示。
- 输入重复次数可设置当前帧重复多少次，点击\*可应用到当前level的全部帧。

#### 对象编辑

- 点击矩阵中的对象，或点击对象列表中的对象，选中对象。
- 点击“左转90”或“右转90”旋转对象。
- 点击“修改基准”后选择新的基准点，再点击“确认”。
- 点击 `+`、`-` 调整对象层级；格子右上角的数字代表此格子有多少重叠对象；绿色对象仍具有显示优先权。
- 点击“改色”将对象修改为当前画笔颜色。
- 点击“复制前帧”“复制后帧”或“复制全帧”，将选中对象复制到其他帧。
- 点击“绿色到全帧”“蓝色到全帧”或“粉色到全帧”，将当前帧对应颜色的对象复制到其他帧。
- 点击“删除”删除选中对象；删除前会显示确认提示。

#### 对象

显示当前帧中的对象列表、颜色、位置、层级和占用格数。

点击列表中的对象即可选中它。

#### 画笔

- 点击“新增对象”或“选择/移动”切换编辑模式，快捷键Q。
- 点击 Color 0、Color 1、Color 2 或 Color 3 选择当前画笔颜色。
- 当前区域显示画笔颜色、矩阵尺寸、缩放比例和对象数量。
- 点击“启动游戏”快速启动当前 Simple 游戏，用于验证运行中的玩法。
- 点击“停止游戏”停止运行。
- 点击“打开预览”播放当前关卡的只读帧序列，显示当前帧和重复次数，不会启动游戏或打开 Debug Panel。
- 点击“校验”检查当前编辑内容。
- 点击“保存”保存游戏数据。
- 点击“导出 GIF”导出当前关卡的 GIF 文件。

## 帮助

### ELC-408 调试助手

打开顶部“帮助”下拉菜单，点击“调试助手”。调试助手包含三个 Tab：配置、布线工具和调试工具，目前能够调通单控制器，单接收器，后续尝试多控制接收器。

#### 配置

- 设置 TCP 服务端口、重复按下开关和防抖时长。
- 从下拉框选择 SDK 使用的网卡。
- 选择控制器型号：HC08 或 HC04。
- 选择 RGB 线序。
- 点击“下载 conf 文件”生成配置 JSON 文件。
- 修改配置文件后，点击“重新加载配置”让应用重新读取配置。

#### 布线工具

- 设置画面宽度、画面高度和每通道最大点数。
- 选择“逐行优先”“逐列优先”“行蛇形”或“列蛇形”。
- 左键点击 Canvas 添加点位。
- 左键拖拽框选区域，系统会按照当前布线模式排序添加未占用点位。
- 右键点击已点亮点位删除点位。
- 右键拖拽框选区域，删除区域内的点位。
- 查看通道列表和每个通道的点位数量。
- 点击“下载 wiring 文件”生成 wiring JSON 文件。

#### 调试工具

- 选择 RGB 线序、网卡和控制器型号。
- 点击“搜索”搜索已连接的控制器。
- 设置控制器数量、每通道最大点数、显示颜色和帧间隔。
- 点击“开始”开始发送调试报文，点击“停止”结束发送。
- 在“协议日志”中查看发送和接收报文的时间、方向、类型和十六进制内容。
- 对较长报文点击展开按钮查看完整内容，再次点击收起。
- 点击“清空日志”清除当前协议日志。

## 进入游戏

“进入游戏”用于完整的玩家游戏流程，包括待机、游戏配置、开始游戏、游戏运行和结束处理。

该入口仍在开发中，当前主要用于验证完整游戏生命周期和 Touch 界面交互，后续拆分为玩家端和副屏端。
